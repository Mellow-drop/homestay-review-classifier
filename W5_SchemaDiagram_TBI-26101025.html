Relationships & Data Flow:
The database uses a One-to-Many relational structure. A single Session acts as a batch container that holds multiple Classified Reviews. The session_id Foreign Key (🔗) links each review strictly to its parent session. If a Session is deleted, a cascading deletion rule instantly removes all associated child reviews, guaranteeing data integrity and preventing orphaned records.
